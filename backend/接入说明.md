# ✅ OpenAI Realtime / OpenRouter 接入说明

> EN: Integration notes for OpenAI Realtime (voice) and OpenRouter (text).  
> EN: This file is kept in Chinese; English annotations are added inline for US handoff.

本项目支持两种模型接入：

> EN: This project supports two provider modes:

- **OpenAI Realtime**：语音直连（低延迟、可打断）
- **OpenRouter Chat**：文本对话（适合仅文本学习）

> EN:
> - OpenAI Realtime: direct voice streaming (low latency, supports barge-in / interruption)
> - OpenRouter Chat: text-only chat (useful when you only want text learning)

## 🚀 启动方式

> EN: How to start.

```bash
cd backend
npm install
npm start
```

看到以下输出表示成功：
```
✅ Realtime 音频流服务器运行在 ws://localhost:3002
```

> EN: When you see the log above, the server is running successfully.

## 🔑 密钥配置

> EN: API key configuration.

请使用 `设置API密钥.bat` 写入 `backend/.env`，或手动在 `.env` 中设置。

> EN: Use `设置API密钥.bat` to write `backend/.env`, or edit `.env` manually.

至少配置一种：

- OpenAI（语音 + 文本）：`OPENAI_API_KEY`
- OpenRouter（文本）：`OPENROUTER_API_KEY`

> EN: Configure at least one: `OPENAI_API_KEY` (voice + text) or `OPENROUTER_API_KEY` (text).

可选配置：

- OpenAI：`OPENAI_API_BASE`（可带或不带 `/v1`）、`OPENAI_REALTIME_URL`
- OpenRouter：`OPENROUTER_API_BASE`（默认 `https://openrouter.ai/api`）、`OPENROUTER_MODEL`

> EN:
> - OpenAI: `OPENAI_API_BASE` (with or without `/v1`), or directly set `OPENAI_REALTIME_URL` for a fixed WS endpoint
> - OpenRouter: `OPENROUTER_API_BASE` (default `https://openrouter.ai/api`), `OPENROUTER_MODEL`

### OpenRouter 示例（推荐文本模式）

> EN: Example `.env` for OpenRouter (recommended for text-only mode).

在 `backend/.env` 中配置：

```bash
OPENROUTER_API_KEY=sk-or-你的OpenRouter密钥
OPENROUTER_API_BASE=https://openrouter.ai/api
OPENROUTER_MODEL=openai/gpt-audio-mini

> EN: Replace `sk-or-你的OpenRouter密钥` with your real OpenRouter key.
```

### GPTsAPI 中转示例（可选）

> EN: Optional example for using a relay/proxy base URL (GPTsAPI).

在 `backend/.env` 中配置：

```bash
OPENAI_API_KEY=sk-你的GPTsAPI密钥
OPENAI_API_BASE=https://api.gptsapi.net/v1

> EN: Replace `sk-你的GPTsAPI密钥` with your real key.
```

如果中转要求固定 WebSocket 地址，可使用：

> EN: If your relay requires a fixed WebSocket URL, set `OPENAI_REALTIME_URL` explicitly:

```bash
OPENAI_REALTIME_URL=wss://api.gptsapi.net/v1/realtime?model=gpt-4o-realtime-preview-2024-12-17
```

快速验证（将 `$OPENAI_API_KEY` 换成环境变量或直接替换为密钥）：

> EN: Quick connectivity check (replace `$OPENAI_API_KEY` with your env var or paste the key directly).

```bash
curl https://api.gptsapi.net/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
        "model": "gpt-4o",
        "messages": [
          {"role":"user","content":"你好，测试连通性"}

> EN: The sample user message means: "Hi, testing connectivity."
        ]
      }'
```

## 🐛 常见问题

> EN: Common issues.

### 1. API调用失败（401错误）
- 检查API Key是否正确
- 确认API Key是否有效

> EN: 401 = invalid/expired key (check correctness and activation).

### 2. 请求频率限制（429错误）
- 免费额度可能已用完
- 等待一段时间后重试
- 考虑升级到付费版本

> EN: 429 = rate limit / quota exceeded (wait, retry, or upgrade).

### 3. 网络连接失败
- 检查网络连接
- 确认可以访问 OpenAI 的 API 域名
- 检查防火墙设置

> EN: Network errors: verify connectivity, domain reachability, and firewall rules.
