# Blackboard AI 阿里云服务器部署说明

本文说明如何将 Blackboard AI 前端 + 后端部署到一台阿里云 ECS 上，通过 Nginx 提供 HTTPS 并反向代理到 Node 服务。

---

## 一、前置条件

- 已购买阿里云 ECS（建议 1 核 2G 起），系统推荐 **Ubuntu 22.04** 或 **CentOS 7/8**。
- 已绑定域名（可选，也可先用公网 IP 访问；若用 IP，部分浏览器可能限制麦克风等能力）。
- 本仓库已能在本地正常运行（`backend` 配好 `.env`，`npm start` 正常）。

---

## 二、服务器基础环境

### 1. 安装 Node.js（推荐 18+）

**Ubuntu / Debian：**

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
node -v   # 应 >= 18
```

**CentOS：**

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
node -v
```

### 2. 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt-get update && sudo apt-get install -y nginx

# CentOS
sudo yum install -y nginx
```

### 3. 安装 PM2（用于常驻 Node 进程）

```bash
sudo npm install -g pm2
```

---

## 三、上传代码并配置后端

### 1. 上传项目到服务器

任选其一：

- 使用 **Git**（推荐）：在服务器上 `git clone <你的仓库地址>`，或本机 `git push` 后服务器 `git pull`。
- 使用 **SFTP / 宝塔 / 文件管理器**：将整个项目目录（含 `backend`、`front end`）上传到例如 `/var/www/blackboard-ai`。

目录结构建议：

```
/var/www/blackboard-ai/
├── backend/
│   ├── .env          # 在服务器上新建，不要从本机上传含密钥的 .env
│   ├── package.json
│   ├── realtime-audio-server.js
│   └── ...
└── front end/
    ├── index.html
    ├── ai-voice-call.html
    └── ...
```

### 2. 在服务器上创建 `backend/.env`

**不要**把本机的 `backend/.env` 直接上传（避免泄露）。在服务器上新建：

```bash
cd /var/www/blackboard-ai/backend
nano .env
```

内容示例（按需保留/修改）：

```bash
# 至少配置其一
OPENAI_API_KEY=sk-你的OpenAI密钥
OPENROUTER_API_KEY=sk-or-你的OpenRouter密钥

# 可选
OPENROUTER_MODEL=deepseek/deepseek-chat
# PORT 可不写，默认 3002；若与 Nginx 冲突可改为其他
PORT=3002
```

保存后确认 `.env` 未被 Git 跟踪（项目根目录已有 `.gitignore` 忽略 `.env`）。

### 3. 安装依赖并测试后端

```bash
cd /var/www/blackboard-ai/backend
npm install
npm start
```

看到类似 “Realtime 音频流服务器运行在 ws://0.0.0.0:3002” 即表示成功。`Ctrl+C` 停止，下一步用 PM2 常驻运行。

### 4. 使用 PM2 常驻运行后端

在 `backend` 目录已提供 `ecosystem.config.cjs`，可直接使用：

```bash
cd /var/www/blackboard-ai/backend
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup   # 按提示执行命令，实现开机自启
```

常用命令：

```bash
pm2 status        # 查看状态
pm2 logs          # 查看日志
pm2 restart all   # 重启
```

## 四、Nginx 配置（静态站 + 反向代理 /api）

### 1. 创建站点配置

```bash
sudo nano /etc/nginx/sites-available/blackboard-ai
```

若无 `sites-available`（如 CentOS），可用：

```bash
sudo nano /etc/nginx/conf.d/blackboard-ai.conf
```

下面配置假设：

- 网站根目录：`/var/www/blackboard-ai/front end`
- 后端 Node 端口：`3002`
- 域名：`your-domain.com`（若暂时用 IP，将 `server_name` 改为 `_` 或你的公网 IP）

**未配置 SSL 时（仅 HTTP，仅用于测试）：**

```nginx
server {
    listen 80;
    server_name your-domain.com;   # 或 _ 表示默认

    root /var/www/blackboard-ai/front end;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API + WebSocket 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:3002/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # WebSocket 支持
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }
}
```

**已配置 SSL 时（推荐生产环境）：**

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate     /path/to/fullchain.pem;   # 证书路径
    ssl_certificate_key /path/to/privkey.pem;

    root /var/www/blackboard-ai/front end;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3002/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }
}
```

请将：

- `your-domain.com` 改为你的域名或 `_`
- `root` 改为你实际存放 `front end` 的路径
- SSL 证书路径改为你实际证书路径（见下文「配置 HTTPS」）

### 2. 启用配置并重载 Nginx

```bash
# Ubuntu/Debian（若用 sites-available）
sudo ln -s /etc/nginx/sites-available/blackboard-ai /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

```bash
# CentOS 或仅 conf.d
sudo nginx -t
sudo systemctl reload nginx
```

## 五、配置 HTTPS（推荐）

### 方式一：Let's Encrypt（免费）

```bash
sudo apt-get install -y certbot python3-certbot-nginx   # Ubuntu
sudo certbot --nginx -d your-domain.com
```

按提示选择域名并自动配置 Nginx。证书路径一般为 `/etc/letsencrypt/live/your-domain.com/`，在 Nginx 里填写 `fullchain.pem` 和 `privkey.pem`。

### 方式二：阿里云 SSL 证书

在阿里云控制台申请免费 DV 证书，下载 Nginx 版本，上传到服务器（如 `/etc/nginx/ssl/`），在 Nginx 的 `ssl_certificate` / `ssl_certificate_key` 中指向对应文件。

## 六、防火墙与安全

- **只开放 80、443**，不要对公网开放 3002：

```bash
# Ubuntu (ufw)
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable

# CentOS (firewalld)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

- API Key 只存在于服务器上的 `backend/.env`，不要提交到 Git，不要写在 Nginx 或前端代码里。
- 若需限制访问来源，可在 Nginx 中按 IP 或域名做 `allow/deny`，或后续加鉴权。

---

## 七、前端与后端对应关系

- 前端已做「生产环境默认同源 `/api`」：当页面不是 `localhost` / `127.0.0.1` 时，会自动使用 `当前域名 + /api` 作为接口地址和 WebSocket 地址，无需在页面里再填后端地址。
- 后端通过 `PORT` 环境变量或默认 3002 监听，Nginx 将 `https://your-domain.com/api/xxx` 转发到 `http://127.0.0.1:3002/xxx`，WebSocket 同样通过 `/api` 代理到后端。

---

## 八、部署后自检

1. **静态页**：浏览器访问 `https://your-domain.com`（或 `http://公网IP`），应打开首页。
2. **健康检查**：访问 `https://your-domain.com/api/health`，应返回 JSON（含 `ok: true` 等）。
3. **语音/黑板**：打开「AI 语音通话学习模式」页，确认能创建黑板、发消息、语音（若已配 OpenAI Key）。浏览器控制台不应出现跨域或 404。

若 `/api/health` 502，请检查：`pm2 status` 是否在运行、端口是否为 3002、Nginx `proxy_pass` 是否为 `http://127.0.0.1:3002/`。

---

## 九、简要清单

| 步骤           | 说明 |
|----------------|------|
| 安装 Node / Nginx / PM2 | 见第二节 |
| 上传代码       | Git 或 SFTP 到 `/var/www/blackboard-ai` |
| 服务器上建 `.env` | 仅在后端目录，不提交、不上传本机密钥 |
| 后端 `npm install && pm2 start ecosystem.config.cjs` | 常驻 + 开机自启 |
| Nginx 配置     | 根目录指向 `front end`，`/api/` 反代到 3002，并支持 WebSocket |
| 开放 80/443、配置 HTTPS | 推荐 Let's Encrypt 或阿里云证书 |
| 访问首页与 `/api/health` | 确认正常后再对外分享链接 |

按以上步骤即可在阿里云单机上完成 Blackboard AI 的完整部署。若你使用宝塔面板，也可在「网站」里添加站点并手动配置「反向代理」到 `127.0.0.1:3002`，逻辑与上述 Nginx 一致。
